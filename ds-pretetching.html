Prefetching

  • sibling of caching
  • crucial for streaming data over Internet
  • rising star as bandwidth improves, but latency stagnates

Prefetching is About Spatial Locality

  • caching was about temporal locality
    • often correct, sometimes wrong
  • prefetching makes a different bet
    • often wrong, sometimes correct

Big Picture

  • “perfecting” → initiating fetch of data in advance of need
    • trivial example is file system read-ahead
    • hoarding in Coda is at opposite extreme time scale
  • Completing of prefetch may occur before or after need occurs
    • before → complete masking of access latency
    • after → reduced effective latency
  • Impacts on cache
    • commitment of critical resources based on imperfect knowledge
    • erroneous commitment may increase latency
    • demand request delayed behind prefetching latency
  • Fast cheap abort important to success
    • often difficult to ensure
    • exactly when prefetching is most useful

The Perils of Prefetching (imperfect knowledge)

Delay-bandwidth Product

  • Prefetching most valuable when high latency & high bandwidth
    • early knowledge + resources to act on that knowledge
    • worst case: long skinny pipe (knowledge alone doesn’t help)
    • prefetching safer with higher bandwidth
    • prefetching more valuable with high latency

Why Important in the Long Run

  • Caching and prefetching only known techniques for reducting latency
  • latency is the killer, not bandwidth
    • fundamentally, more difficult
    • bandwidth improved through parallelism
    • software and systems trends increase latency
      • increase use of layers
      • security technologies
  • High hit ratio → only misses are initial references (cold misses)
    • prefetching is the only way to eliminate these

Knowledge for Perfecting

  • Sources of prefetching hints
    • reference stream
    • explicit user hints
    • explicit programmer hints
    • compiler extraction of hints from source code
    • historical observation (previous execution)
    • higher-level context awareness
      • only calendars, email analysis, crowd behavior analysis

Timing is Everything

  • if cache pace is plentiful
    • no replacements
    • cold cache filling is all that matters
    • no harm is prefetching as early as possible
    • only resource that is scare is network bandwidth (and server)
    • only harm: queueing delay for demand misses, bandwidth wasted
  • if cache space is scarce
    • replacements is frequent
    • prefetching too early → evict before first use
  • prefetching too late
    • data is not yet in cache when demand access arrives
    • may reduce (but not eliminate) cold cache miss delay

Speculation

  • prefetching is a form of data speculation
  • Data staging is another example of data speculation
    • prefetch to a nearby cache rather than to local cache
    • you may not have connectivity to that cache but will “soon”
  • complement of data speculation is speculative execution
