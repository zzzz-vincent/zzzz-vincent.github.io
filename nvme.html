1. NVMe for beginners

Storage Interface at High-level

  • An SSD runs a COMMAND SET over TRANSPORT
    • AHCI over SATA
    • SCSI over SAS
    • NVMe over PCIe (Local) or FC/RoCE (remote)
  • Low-level “APIs” to read/write data from/to the device
  • The storage interface defines the ways to interact with the storage devices
    • Device management: probing device capability, mark bad blocks/sectors
    • Read/Write requests in specific formats
    • Defined as protocols which specifies the steps for it to work (eg. invalid request types)

Non-Volatile Memory Express (NVMe)

  • NVMe is a “software” protocol which runs on top of PCIe bus (bus standard)
  • NVMe is a scalable interface to connect SSDs and the host system
  • NVMe inherits the benefits of PCIe bus
    • High bandwidth (e.g., 1GB/s)
    • Low latency (e.g., 1us)
    • Parallelism (multi-lanes, multi- inflight requests)
    • Compatibility: industry standard supported by many vendors
  • Support massive parallel I/O queues for submissions/completions
    • 64K I/Os per queue, Up to 65536 queue pairs. (Legacy SAS and SATA can only support single queues and each can have 254 & 32 entries respectively)
    • The NVMe host software can create queues, up to the maximum allowed by the NVMe controller, as per system configuration and expected workload
    • This architecture allows applications to start, execute, and finish multiple I/O requests simultaneously and use the underlying media in the most efficient way to maximize speed and minimize latencies
  • NVMe SSD is have small overhead compared with SATA SSD
  • NVMe Gets the Most Performance from Multicore Processors
    • NUMA-optimized protocol: This allows for multiple CPU cores to share the ownership of queues, their priority, as well as arbitration mechanisms and atomicity of the commands

Message Queue / Request Queue

  • As in cloud computing, the message queue is one such way to allow various microservices to exchange and process application data asynchronously and hence improve performance, reliability, and scalability
  • In NVMe queue, the compute nodes, which own the overall applications as well as underlying microservices, exchange data asynchronously with each other through message queues and perform IO operations with their respective storage devices
  • With longer NVMe IO queues, the storage devices can queue IOs and fully utilize all available parallel channels to access underlying storage media and complete IOs as fast as possible
  • NVMe protocol supports out of order processing, the storage devices can re-order IOs and further improve IO performance.

NVMe Concepts

  • The host view of an PCIe/NVMe controller
    • PCIe Base Address Register (BAR)
    • Read/Write BAR to program/control it
  • NVMe protocol
    • Queue Pair (QP)
      • Submission Queue (SQ)
      • Completion Queue (CQ)
    • Control/Data Separation
      • Admin QP
      • I/O QP
      // map NVMe BAR to the OS/driver space
void *nvme_ctrl = mmap(base_addr, len, …); 
// query information about one register
read(nvme_ctrl + reg_offset, 4); 
// program the controller to do something
write(nvme_ctrl + reg_offset, 8);

How does NVMe represent an I/O?

  • The host sends NVMe commands down to the SSD
  • The SSD returns NVMe completions to the host
  • NVMe Command Format (64 bytes)
    • Opcode: read/write/flush
    • Command Identifier: unique ID
    • Data Pointer: Point to the buffer in host DRAM
  • NVMe Completion Format (16 Bytes)
    • Which SQ entry does this completion entry correspond to?
      • Submission Queue Identifier (SQID)
      • Command Identifier (CID)
    • What’s the results of command execution? Success or failure?
      • Stat`us Field (SF)
    • Submission Queue Head Pointer (SQHD)

NVMe Queue

  • NVMe commands/completions are put into the NVMe queues
  • NVMe Queues: A shared circular buffer
    • In Host DRAM or SSD DRAM
    • Producer/consumer model
    • Used by the host to submit commands
    • Used by the controller to post completions
    • Coordination between the host and SSD to maintain the status of the queues

Host/SSD Notification

  • How does the SSD know there are commands to process?
    • The host notifies the controller of new submission by ringing Doorbell (DB) Register → Write SQ Tail Position to the register
  • How does the host know whether a request has completed?
    • The controller may notify the host of completions with interrupts

MVMe I/O Processing Flow

  • Controller executes commands Out-of-Order
  • NVMe minimizes Memory Mapped I/O (MMIO) operations
    • Allowing the host to enqueue multiple commands at a time
    • Allow the host to process multiple completions at a time
